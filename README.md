Project Description & Purpose
This project is a responsive portfolio website template designed to be easily forked and customized by developers, designers, or creatives who want a clean starting point for their personal portfolio. The template focuses on:
A clear content structure
Lightweight performance
Easy customization without build tools or frameworks
The template showcases a graphic designer, her work, as well as some of her hobbies, but all text, images, and sections are placeholders and are meant to be adapted to the user‚Äôs own profession, projects, and personal brand.

Tech Stack
This project is built using vanilla web technologies, keeping everything simple, accessible, and dependency-light:
HTML5 ‚Äì Semantic markup and content structure
CSS3 ‚Äì Layout, styling, and responsive design
JavaScript (Vanilla) ‚Äì Animations and interactive behavior
ImageKit CDN ‚Äì Optimized image delivery for better performance

Setup Instructions (Local Development)

Follow these steps to run the project locally on your machine:
1. Fork the Repository
Click the Fork button in the top-right corner of the GitHub repository page.
This creates a copy of the project under your own GitHub account, allowing you to modify it freely.
2. Clone Your Fork
On your forked repository page, copy the repository URL and clone it to your local machine:
git clone https://github.com/ValeMtzAlvarez/portfolio-template-2025.git
Navigate into the project folder:
cd repository-name
3. Open the Project in Visual Studio Code
Open the folder using Visual Studio Code:
Right-click the folder and select Open with Code, or

4. Run the Project in the Browser
Since this is a static project, no build step is required.
You can choose one of the following options:
Option A ‚Äî Open directly
Locate the index.html file
Double-click it or drag it into your browser
Option B ‚Äî Use Live Server (Recommended)
Install the Live Server extension in VS Code
Right-click index.html and select ‚ÄúOpen with Live Server‚Äù

5. Start Customizing
Once running, you can begin editing:
HTML files for content and structure

JavaScript files for animations and interactions
No additional dependencies, package installations, or build tools are required.


Customization Guide
This template is designed to be easily personalized:
1. Text Content
Edit the text directly in the HTML files
Replace placeholder sections (about, projects, contact, what else do i do, etc.) with your own information
2. Images
Replace image URLs with your own
Images are served via ImageKit CDN, so you can:
Upload your images to ImageKit
Replace the existing URLs with your own optimized links
3. Styling
Modify colors, fonts, and layout in the CSS files
The structure is kept simple to avoid unnecessary overrides
4. Animations
JavaScript animations are written in plain JavaScript
You can adjust timings, triggers, or remove animations as needed

Reflection

Overall, I am very proud of the website I designed and developed. The project challenged me on both a technical and creative level, and although I encountered several difficulties throughout the process, I am satisfied with the final outcome.
Some of the biggest challenges I faced were related to more advanced interactions and motion, such as implementing parallax scrolling, working with scroll-based reveal animations, and exploring GSAP. While these aspects were particularly exciting to me at the beginning of the project, they also proved to be more complex than expected. Accessibility, was another area that required careful consideration and added complexity to the animation process.
As the deadline approached, I made a conscious decision to prioritize stability and clarity within the code, and overall design quality over adding complex animations that might compromise the look of the website, as the idea is not to incorporate elements just for decoration‚Äôs sake. Further, the animations that are present are subtle and do not significantly impact usability, which is why I did not consider it necessary to implement a prefers-reduced-motion media query.
Despite the challenges I faced, I believe the project successfully reflects my personal creative style. I enjoyed experimenting with color combinations and slightly chaotic and asymmetrical layouts, which allowed me to move away from more rigid portfolio designs and create something more expressive. Also, I believe it‚Äôs always appreciated when someone shares more about themselves and not just their work, as it offers a deeper insight into who they are, which is why I decided to include a section on hobbies and interests. This experimentation helped me leave a clear personal and creative mark on the project.
In conclusion, this project was a valuable learning experience. It helped me better understand my current technical limits, the importance of prioritization, and the value of adapting ambitions to what is realistically achievable. Also, it reinforced the idea that simplicity can be a strong and intentional design choice. Choosing not to overcomplicate interactions or visuals does not mean a lack of effort, ambition, or creativity. On the contrary, simplifying the final result required making conscious decisions about what truly added value to the website and what could be removed without affecting its purpose.
